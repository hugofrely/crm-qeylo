# Chat Conversations History — Design

## Summary

Add conversation history to the CRM chat: users can have multiple separate conversations, navigate between them via a right-side sidebar, start new conversations, rename and delete them. Titles are auto-generated by the LLM after the first exchange.

## Architecture

### New Model: `Conversation`

```python
class Conversation(models.Model):
    id = models.UUIDField(primary_key=True, default=uuid.uuid4)
    organization = models.ForeignKey(Organization, on_delete=models.CASCADE)
    user = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.CASCADE)
    title = models.CharField(max_length=200, default="Nouvelle conversation")
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        ordering = ["-updated_at"]
```

### Modified Model: `ChatMessage`

- Add `conversation = models.ForeignKey(Conversation, on_delete=models.CASCADE, related_name="messages")`
- Nullable during migration, then made required

## API Endpoints

| Method | URL | Description |
|--------|-----|-------------|
| GET | `/api/chat/conversations/` | List conversations (title, dates, last message preview) |
| POST | `/api/chat/conversations/` | Create new conversation |
| DELETE | `/api/chat/conversations/<id>/` | Delete conversation (cascade messages) |
| PATCH | `/api/chat/conversations/<id>/` | Rename conversation |
| GET | `/api/chat/conversations/<id>/messages/` | Messages for a conversation |
| POST | `/api/chat/stream/` | Send message (requires `conversation_id` param) |

### Sidebar Conversations (right side)
- List sorted by `updated_at` desc
- Each item: title + relative date
- "+" button for new conversation
- Hover: rename / delete buttons
- Active conversation highlighted
- Responsive: collapses on small screens

## User Flow

1. **First visit**: a conversation is created automatically
2. **"+ New conversation"**: POST creates conversation, switch to it, empty chat with welcome message
3. **First message sent**: after AI response, title is auto-generated via LLM
4. **Switch conversation**: click in sidebar, loads its messages
5. **Delete**: confirmation dialog, DELETE, switch to most recent

## Title Generation

After the first assistant response in a conversation, a separate LLM call generates a short title (5 words max) based on the first user+assistant exchange. Updated via PATCH.

## Data Migration

1. Create `Conversation` model
2. Add nullable `conversation` FK on `ChatMessage`
3. Data migration: group existing messages by `(user, organization)` into one "legacy" conversation each
4. Make FK non-nullable

## AI Context

Only messages from the current conversation are sent to the LLM as history. This improves response relevance by avoiding topic pollution across conversations.
